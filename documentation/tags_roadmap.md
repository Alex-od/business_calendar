# План: теги (Radiant Days)

Документ фиксирует реализованное поведение тегов и отложенные улучшения. Обновлять по мере реализации.

**Статус:** теги реализованы (БД v10, many-to-many, цвет в UI). Ниже — **оставшийся бэклог**.

---

## Реализовано (2026-06-04)

- Папки переименованы в **теги**; у заметки **несколько** тегов (`tagGuids`, таблица `note_tags`).
- Виртуальный фильтр **«Без тегов»** (заметки без строк в `note_tags`).
- Экран **«Теги»**, список заметок по тегу, мультивыбор чипов в форме заметки.
- Быстрый доступ: закреплённые теги + **«Ещё»** для остальных.
- **Цвет тега** (2026-06-04): поле `color` (`EventColor`), миграция **9→10**, палитра при создании/редактировании, цветные чипы и точки в списке тегов.

## Заметки по реализации (черновик)

### 1. Фильтр календаря

- Состояние выбранного тега (один guid или `Tag.UNTAGGED_GUID`) в `CalendarUiState`.
- Чипы/меню в тулбаре календаря; при смене тега — перезагрузка событий месяца/дня с учётом фильтра.
- Не путать с мультивыбором тегов **на заметке** — фильтр только для **просмотра** календаря.

### 2. WebSocket bridge

- В объект заметки в `result[]` добавить `"tag_guids": ["guid1", "guid2"]` (массив строк, может быть пустым).
- Обратная совместимость: не ломать клиентов, которые поле игнорируют.
- Обновить примеры в `documentation/ws_bridge_server.md`.
